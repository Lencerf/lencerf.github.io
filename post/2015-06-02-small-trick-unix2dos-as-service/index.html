Make unix2doc a system service of OS X

June 02, 2015 Technique OS X , Automator

Revising academic papers is really a hard work. What makes things worse is that WinEdit used by my mentor is only able to open .tex files with CRLF line breaks while TeXShop.app uses LF as a line ending.

Unix, on which OS X is based, uses LF as the default line break, different from Windows. Although I can use TextWrangle.app to convert text files between the two formats, but I really don’t like TextWrangle. (Sublime Text is wonderful!!!)

So I find unix2dox, a command line tool that is developed to “convert text between DOS, UNIX, and Mac formats”. With Automator, the problem can be better solved.

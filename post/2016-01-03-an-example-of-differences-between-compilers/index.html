An example of differences between compilers

January 3, 2016
Technique C

For the code fragment below,

//ch1.c
#include "stdio.h"
int main(int argc, char const *argv[]) {
    int a = 1;
    int c = (a=a * 2) + (a = a + 3);
    printf("a = %d, c = %d.\n", a, c);
    return 0;
}

if gcc is used to compile it, there are no errors or warnings. Run the binary file and we will get:

a = 5, c = 10.

However, if I use clang to compile the codes, there will be a warning:

ch1.c:6:15: warning: multiple unsequenced modifications to ‘a’ [-Wunsequenced]

Run the generated file:

a = 5, c = 7.

Personally, I think clang is better than gcc in dealing this problem. But why I, the programmer, write such unreadable codes?!
